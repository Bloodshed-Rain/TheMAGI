# Character Wireframes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the unused emoji decoration on the Characters page with character-tinted wireframe imagery for all 26 characters in the roster.

**Architecture:** Optimize the existing `site/*_wireframe.png` files (white-line-on-transparent PNGs) into a smaller asset set committed under `src/renderer/assets/characters/`. Each character tile and detail-page hero renders a `<div>` whose `mask-image` is the wireframe and whose `background-color` is the character's accent color from `CHARACTER_META`. This produces a tinted silhouette without requiring `<img>`/CSS-filter tricks.

**Tech Stack:** React 19, Vite 7, TypeScript (strict + `exactOptionalPropertyTypes`), CSS (vanilla, in `src/renderer/styles/components.css`), Sharp (new devDependency, used only by the one-shot optimization script).

**Spec reference:** `docs/superpowers/specs/2026-04-25-character-wireframes-design.md`

**Note on git:** This project is not a git repository. Tasks omit `git commit` steps. The "frequent commits" cadence is replaced with logical checkpoint pauses where the engineer should verify the previous step before moving to the next.

**Note on tests:** The vitest suite in this project covers pipeline / config / db / importer logic, not React UI. There is no React Testing Library setup. Verification for UI tasks below is therefore: TypeScript compile, Vite build, and a documented manual visual check. Where automated checks are meaningful (asset size budgets, file existence) they are included.

- [ ] **Step 1.2: Create the optimization script**

Create `scripts/optimize-wireframes.mjs` with this exact content:

```javascript
// One-shot optimizer for character wireframe PNGs.
// Reads from site/*_wireframe.png, writes optimized files into
// src/renderer/assets/characters/<short-key>.png.
// Run manually: `node scripts/optimize-wireframes.mjs`
import sharp from "sharp";
import { mkdir, stat } from "node:fs/promises";
import { join, dirname } from "node:path";
import { fileURLToPath } from "node:url";

const __dirname = dirname(fileURLToPath(import.meta.url));
const SRC_DIR = join(__dirname, "..", "site");
const OUT_DIR = join(__dirname, "..", "src", "renderer", "assets", "characters");

// short-key (filename without .png) -> source filename in site/
const MAP = {
  fox: "fox_wireframe.png",
  falco: "falco_wireframe.png",
  marth: "marth_dair_wireframe.png",
  sheik: "sheik_wireframe.png",
  falcon: "captain_falcon_wireframe.png",
  puff: "jigglypuff_wireframe.png",
  peach: "peach_wireframe.png",
  ics: "ice_climbers_wireframe.png",
  samus: "samus_wireframe.png",
  pikachu: "pikachu_wireframe.png",
  luigi: "luigi_wireframe.png",
  mario: "mario_wireframe.png",
  doc: "dr_mario_wireframe.png",
  yoshi: "yoshi_wireframe.png",
  ganon: "ganondorf_wireframe.png",
  link: "link_wireframe.png",
  ylink: "young_link_wireframe.png",
  zelda: "zelda_wireframe.png",
  roy: "roy_wireframe.png",
  mewtwo: "mewtwo_wireframe.png",
  gnw: "game_and_watch_wireframe.png",
  ness: "ness_wireframe.png",
  bowser: "bowser_wireframe.png",
  kirby: "kirby_wireframe.png",
  dk: "donkey_kong_wireframe.png",
  pichu: "pichu_wireframe.png",
};

await mkdir(OUT_DIR, { recursive: true });

let total = 0;
const results = [];
for (const [outName, srcName] of Object.entries(MAP)) {
  const srcPath = join(SRC_DIR, srcName);
  const outPath = join(OUT_DIR, `${outName}.png`);
  await sharp(srcPath)
    .trim()                                          // crop transparent borders
    .resize({ width: 800, withoutEnlargement: true })// shrink to max 800px wide
    .png({ compressionLevel: 9, palette: true, quality: 80, effort: 10 })
    .toFile(outPath);
  const { size } = await stat(outPath);
  total += size;
  results.push({ outName, kb: size / 1024 });
}

results.sort((a, b) => b.kb - a.kb);
for (const r of results) console.log(`${r.outName.padEnd(8)}  ${r.kb.toFixed(1)} KB`);
console.log("------------------------------");
console.log(`Total: ${(total / 1024 / 1024).toFixed(2)} MB across ${results.length} files`);
```

- [ ] **Step 3.1: Replace the image map and component (lines 16–29)**

Delete lines 16–29 of `src/renderer/pages/Characters.tsx`:

```tsx
// ── Character card art (dynamic, falls back to emoji) ────────────────

const CHARACTER_IMAGE_NAMES: Record<string, string> = {
  Marth: "marth.png",
  Peach: "peach.png",
};

function CharacterCardImage({ character }: { character: string }) {
  const [error, setError] = useState(false);
  const filename = CHARACTER_IMAGE_NAMES[character];
  if (!filename || error) return null;
  const src = new URL(`../assets/characters/${filename}`, import.meta.url).href;
  return <img src={src} alt="" className="char-card-bg-img" onError={() => setError(true)} draggable={false} />;
}
```

Replace with:

```tsx
// ── Character wireframes (color-tinted via CSS mask) ─────────────────

// Map App short-key -> filesystem short-key (lowercase, ascii-only).
const WIREFRAME_FILE_KEYS: Record<string, string> = {
  Fox: "fox",
  Falco: "falco",
  Marth: "marth",
  Sheik: "sheik",
  Falcon: "falcon",
  Puff: "puff",
  Peach: "peach",
  ICs: "ics",
  Samus: "samus",
  Pikachu: "pikachu",
  Luigi: "luigi",
  Mario: "mario",
  Doc: "doc",
  Yoshi: "yoshi",
  Ganon: "ganon",
  Link: "link",
  YLink: "ylink",
  Zelda: "zelda",
  Roy: "roy",
  Mewtwo: "mewtwo",
  "G&W": "gnw",
  Ness: "ness",
  Bowser: "bowser",
  Kirby: "kirby",
  DK: "dk",
  Pichu: "pichu",
};

// Eagerly load every wireframe asset URL at build time. Vite resolves
// each path to a hashed URL and inlines the map.
const WIREFRAME_URLS = import.meta.glob<string>(
  "../assets/characters/*.png",
  { eager: true, query: "?url", import: "default" }
);

function getWireframeUrl(character: string): string | undefined {
  const fileKey = WIREFRAME_FILE_KEYS[character];
  if (!fileKey) return undefined;
  return WIREFRAME_URLS[`../assets/characters/${fileKey}.png`];
}

function CharacterWireframe({
  character,
  size = "tile",
}: {
  character: string;
  size?: "tile" | "hero";
}) {
  const src = getWireframeUrl(character);
  if (!src) return null;
  const color = CHARACTER_META[character]?.color ?? "var(--accent)";
  const className =
    size === "hero"
      ? "character-wireframe character-wireframe-hero"
      : "character-wireframe character-wireframe-tile";
  return (
    <div
      className={className}
      aria-hidden="true"
      style={
        {
          ["--wf-src"]: `url(${src})`,
          ["--char-color"]: color,
        } as React.CSSProperties
      }
    />
  );
}
```

Note: the `as React.CSSProperties` cast is required because TypeScript's `exactOptionalPropertyTypes` strict mode disallows arbitrary CSS custom-property keys on the typed `style` prop. This is the standard React pattern for setting CSS variables inline.
